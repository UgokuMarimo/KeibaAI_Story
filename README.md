# 🐴 競馬予測AI (KeibaAI) - プロジェクト概要＆運用マニュアル

## 1. プロジェクト概要 (Overview)
### 📊 制作内容
本プロジェクトは、「**根拠を説明できる競馬予測AI (XAI)**」の開発プロジェクトです。
過去数十年分の中央競馬データを学習した機械学習モデル（LightGBM）が、レースの勝ち馬を高精度に予測します。

### 💡 特徴と背景
従来の競馬予測AIは「どの馬が勝つか」という結果のみを出力するものが多く、利用者には予測根拠がブラックボックス化していました。
そこで本プロジェクトでは、**SHAP (Shapley Additive exPlanations)** 技術と **生成AI (Gemini)** を組み合わせることで、「なぜその馬を推奨するのか（例：上がりの速さが評価された、適性がマッチしている等）」を自然言語で分かりやすく説明できるAIを構築しました。

さらに、リアルタイムオッズ情報から期待値（EV）を動的に計算し、基準を満たす馬に自動投票する「**UMACA自動投票システム**」も搭載しています。

```text
│   ├── training/            # モデル学習スクリプト (train_model.py)

```

## 🛠️ 3. 日常の運用手順 (Operation Manual)

### 3-1. 【自動運用】レース当日の自動予測・自動投票の起動
当日の朝、以下のバッチファイルをダブルクリックまたは実行してください。
全レースの出馬表取得 ➡ 予測 ➡ 期待値判定 ➡ UMACA自動投票までを完全自動で処理します。
```bash
run_scheduler.bat
# または
python src/main_scheduler.py
```

### 3-2. 【事後作業】レース結果の更新（答え合わせ）
レースがすべて終了した日の夜に実行します。
実際の「着順」や「払い戻し金」をデータベース（`predictions.db`）に反映させ、成績・回収率を最新に更新します。
```bash
python src/prediction/update_results.py
```

## 🔄 4. モデル更新・再学習手順 (Model Update)
新馬戦や最新レースの結果を学習データに加え、モデルを賢くします。

### 一括自動更新バッチの実行
面倒な「最新データの収集 ➡ 特徴量構築 ➡ 本番モデル再学習 ➡ 成績レポート送信」の全ステップを、**コマンド1つで実行**できます。
```bash
run_update.bat
# または
python src/update_model.py
```
*   **Discord自動通知**：処理の進行状況および各ステップの成否・実行時間は、完了後に自動的にDiscordの「モデル再学習用」チャンネルに通知されます。
*   **特定の日付だけをデータ追加したい場合**：
    ```bash
    python src/update_model.py --date 20260704 20260705
    ```

## 📡 JRA-VAN データベースとネット競馬のハイブリッド設計
*   **リアルタイム予測時**：
    システムの負荷軽減と高速化のため、まずJRA-VAN公式データベース（MySQL: `172.30.0.8`）から出馬表やオッズを取得します。
    MySQLサーバーがオフラインの場合は、**自動的に従来の netkeiba スクレイピングへとフォールバック**され、安全に処理が続行されます。
*   **再学習用データ取得時**：
    EveryDBの同期ラグに左右されないよう、安定した **netkeiba webスクレイピング** を用いて過去のレース結果を取得します。

## 💸 UMACA自動投票の一本化フローについて
自動投票時のオッズ取得と投票処理は、Playwrightブラウザ（1セッションのみ）で効率的に実行されます。
1.  ログイン処理の高速化（カード番号・生年月日の即時入力）
2.  ログイン後、目的のレースの馬番選択画面へ直接遷移
3.  画面上のリアルタイムオッズを取得し、予測勝率から期待値（EV）をその場で判定
4.  購入基準（期待値 $\ge 1.3$）を満たした馬のみ、金額（100円〜）を入力して投票実行（本番 / 模擬）
5.  投票結果をDiscordへ瞬時に通知（通知メッセージに「開催場名（例: **小倉 11R**）」が明記され、視認性が向上しました）

## 🖥️ 5. 統合ダッシュボード (app.py) の機能
Streamlit を用いた統合管理Web UIです。高級感のあるプレミアムダークテーマと直感的な操作パネルを提供します。

### 🔮 リアルタイム予測 ＆ 期待値分析
* **レースカレンダー**: 年月および特定日付から開催スケジュールをロードし、直感的なグリッドUIから各レースの予測画面に即座にアクセスできます。
* **AI予測ランキング**: LightGBMモデルによる1着予測確率からAI予想印（◎, ◯, ▲, △）を自動付与してランキング表示します。
* **詳細なAI解説 (Gemini) ＆ SHAP要因分析**: 
  * 予測上位3頭については、機械学習モデルの判断根拠（SHAP値）を日本語デコードしたデータを基に、Geminiが「勝つための戦術、強みの詳細、懸念リスク、馬券戦略、総評」をプロの予想屋ライクに自動生成します。
  * 特徴量ごとのプラス/マイナス寄与度（SHAP重要度）を可視化したグラフおよび詳細一覧を表示します。
* **期待値(EV)分析ダッシュボード**: 予測勝率とリアルタイム単勝オッズから、期待値に基づいた購入シミュレーションやオッズ乖離を分析します。

### 📈 回収率実績分析
* データベース（`predictions.db`）に蓄積された過去の予測スコアと確定レース結果を基に、馬券シミュレーション（ボックス、上位N頭購入など）を行い、的中率・回収率の推移をビジュアル分析できます。

### 🤖 競馬AIチャット (RAG)
* ベクトルデータベースに蓄積された過去走テキストやレース分析のメタデータを参照しながら、Gemini 2.5 Flashと対話できるチャット機能です。
* 「このレースの傾向は？」「今回のメンバーでのスローペース懸念馬は？」など、直近予測された特定レースや出走馬のコンテキストに沿った鋭い対話が可能です。

### ⚙️ システム管理・データ同期
* データベース内に保存されている総予測件数や結果未同期のレース数などの統計情報を可視化します。
* レース終了後に、確定した着順や配当データをNetkeibaから同期する更新バッチスクリプト（`update_results.py`）をUI上からワンクリックで実行可能です。
