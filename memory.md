## 🛠️ 最新の変更履歴（2026年7月7日実施）

1.  **UMACA自動投票ログインの高速化**：
    *   [umaca_voter.py](file:///c:/keibaAI/src/voting/umaca_voter.py) 内の `login` メソッドを改修。UMACAカード番号と生年月日は `fill()` による即時オートフィル入力とし、暗証番号のみを人間らしいタイピング遅延 (`_human_type`) で入力するように変更。ログイン時間を約3〜4秒短縮。
2.  **Discord投票完了通知の視認性向上**：
    *   [auto_voter_manager.py](file:///c:/keibaAI/src/voting/auto_voter_manager.py) の通知メッセージ作成処理を改修。レースID（例: `202503030111`）から「**中京 11R**」のように開催場・レース番号を自動パースして太字で追加。
3.  **オッズ取得と投票処理のPlaywright一本化 (Phase 2)**：
    *   従来の「オッズ取得用のブラウザ起動」と「投票用のブラウザ起動」の二重起動を廃止。
    *   投票用Playwrightブラウザ（1セッションのみ）でログイン後、UMACAの馬番選択画面へ直接遷移し、そこからリアルタイムオッズを取得 ➡ 期待値（EV）判定 ➡ 対象馬が存在すればそのまま投票金額入力へ進み完了、という一本化フローを実装。
4.  **データベース自動スキーマアップデート（マイグレーション）**：
    *   [db_utils.py](file:///c:/keibaAI/src/utils/db_utils.py) を改修。`predictions` テーブルに `race_class` (クラス) や `race_name` (レース名) カラムが不足している場合に、既存データを破壊せず自動で `ALTER TABLE` を実行して最新のスキーマにアップデートするマイグレーションロジックを追加。
5.  **モデル更新（再学習）結果のDiscord自動通知機能**：
    *   [update_model.py](file:///c:/keibaAI/src/update_model.py) を改修。毎週実行されるモデル更新バッチが、全ステップ終了時（または致命的エラーによる中断時）に、各ステップの成否・実行時間・完了時刻をまとめたレポートをDiscordへ自動送信するように変更。

## 📡 データ取得と予測のアーキテクチャ

*   **リアルタイム予測実行時（直前期待値計算）**：
    *   **ハイブリッド構成**：出馬表や過去走のロード時には、まずJRA-VAN公式データベース（MySQL: `172.30.0.8`）への接続を試みます。
    *   **安全なフォールバック**：MySQLサーバーがオフライン（または接続タイムアウト）の場合は、自動的に従来の netkeiba webスクレイピングに切り替わり、何事もなかったかのように処理を正常終了させます。
*   **モデル再学習（週末バッチ更新）**：
    *   現在、EveryDB2によるMySQLデータベースへのレース結果（成績データ）のインポートが自動化されておらず、手元のデスクトップPC内のデータが更新されていない場合があるため、**従来のネット競馬スクレイピング方式**を採用しています。
    *   `python src/data_collection/scraper_main.py --date YYYYMMDD` を実行することで、指定した日付の全レースのHTMLを [data/raw_html/2026/](file:///c:/keibaAI/data/raw_html/2026/) にキャッシュ保存し、同時に [data/raw/2026.csv](file:///c:/keibaAI/data/raw/2026.csv) に追記します。

## 🚀 今後のAIエージェントへの注意事項

*   **MySQL接続エラー (`172.30.0.8`) について**：
    *   MySQLサーバーは Cloudflare / 社内LAN環境で管理されており、接続が切れている場合は `[DB WARN]` が出ます。これは**自動的にnetkeibaスクレイピングへフォールバックされるため、無理に接続エラーを修復しようとする必要はありません**。
*   **自動投票のモックテスト**：
    *   自動投票の一本化テストは [run_mock_auto_vote_test.py](file:///c:/keibaAI/src/voting/run_mock_auto_vote_test.py) から実行可能です。
    *   引数 `--scenario` (1: 投票成功、2: 期待値低下見送り、3: 上限オッズ超過見送り) を使って動作を簡単にテストできます。
